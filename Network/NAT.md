
当回来的流量到达NAT节点时, 会被抓取, 替换公网的目的地址, 为私网的ip地址
静态NAT
每一个私网地址对应一个公网地址. 无法节省公网地址
使用场景: 多用于内网服务器的映射. 不会用在主机上


# 动态NAT
找一组公网IP作为地址池
问题是, 如果同时上网的主机多余地址池数量的话, 就不行了


NAPT 网络地址端口访问


```shell
# 首先定义一个外网IP的地址族

# 定义一个抓取内网流量的ACL

int g0/0/0
nat outbound 2000 address-group 0 
```

easy IP
不用地址池了, 就用一个公网地址, 然后端口转换来连接外网
瓶颈是并发端口超过65535

nat server
```shell
# 只映射一台设备
```

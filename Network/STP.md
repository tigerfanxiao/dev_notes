生成树协议 801.D 每台交换机都有桥 ID(Bridge ID或 BID), 在生成树网络中来标识不同的桥 ID, 在设备出厂的时候就有. 
桥 ID = 16bit 桥优先级 + 桥MAC 地址
桥 ID 最小的交换机就是根桥, 优先级相同 MAC 地址最小的称为根桥. 根桥具备抢夺性
如何查看交换机的 BID
默认的桥 ID 是 32768
```shell
display stp # CIST Bridge, 每一个接口的开销值

stp mode stp # 启用生成树
stp disable # 关闭生成树
```


### 开销 cost
每一个激活了生成树协议的端口都会计算一个开销值, 计算到达根路径的开销
总体来说带宽越大开销越小, 和双工模式, 端口聚合有关
可以通过命令来修改开销
开销的计算方式
开销有三种计算标准 802.1d-1998, 802.1t, 华为私有标准. 华为设备上默认是 802.1t, 思科的交换机默认的标准是 802.1d

100 兆全双工的端口 802.1d = 18, 802.1t=20000 华为私有=199
1000 兆全双工的端口 802.1d = 4, 华为私有=20

因为开销值是作为链路的开销, 所以修改开销的时候, 要做到链路两遍都要改
dot1t = 802.1t

```shell
# 修改开销标准
stp pathcost-standard dot1d
# 修改开销值
int g0/0/1
stp cost 1000
stp port 100 # 修改接口优先级
```

在现网中, 所有交换机应该要确保使用相同的开销标准

根路径开销 Root Path Cost
一台设备从某个接口到达根桥的 RPC等于从根桥到该设备沿途所有入方向接口的累加

Port ID 
运行生成树的交换机使用 port ID 来标识每个接口, 接口 ID 主要在特定场景下, 选举指定端口
接口 ID = 4bit 接口优先级 + 12bit接口编号. 华为缺省的接口优先级是 128
g0/0/1 的端口编号就是 1, 但是在高端交换机(有多个板卡)可能出现 port id 冲突的情况, 需要到设备上去看


BPDU  Bridge Protocal data unit
通过 802.3 的帧格式来发送. 
BPDU 分为两种
1. 配置 BPDU 完成生成树计算
2. TCN BPDU 拓扑改变通知, 只有在拓扑变化时才会产生

配置BPDU 报文格式
1. PID 协议 ID, 对生成树而言, 总是 0
2. PVI 协议版本号, 对应生成树而言, 总是 0
3. BPDU Type: 有两种 0x00表示配置 BPDU, 0x80表示 TCN BPDU
4. Flags 8bit 生成树使用了最低和最高两个 bit 位. 最低位是 TC 表示拓扑变更, 最高位 TCA 表示拓扑变更确认
5. Router-ID 根桥的桥 ID
6. RPC 根路径开销
7. Bridge ID 发送者的桥 ID, 4096 的倍数,范围在 0-61440
8. Port ID
9. Message Age 本质上是到达根的跳数
10. Max Age 默认是 20s
11. Hello Time 默认是 2s, 根桥发送 BPDU 的间隔
12. Forward Delay 转发延迟, 默认是 15s

根端口是非根桥去往根桥路径最优的端口. 
根端口是接受来自根桥最好 BPDU 的端口
根端口只在非根桥设备上, 且每台设备只有一个
根桥没有根端口

如果一个端口既不是根端口也不是指定端口, 则为预备端口, 预备端口会被阻塞

### 指定端口
在每条链路上选举一个指定端口. 每条链路指每个冲突域, 交换机的一个端口就是一个冲突域
指定端口是用来转发最好的 BPDU 的端口
一般情况下, 根桥的所有端口都是指定端口



STP 选举机制
1. 选择根桥, 全网 BID 最小的交换机为根桥, 具备抢夺性和唯一性
2. 除根桥外, 其他交换机为非根桥
3. 在每个非根桥上选举一个根端口
4. 在每一条链路上选举一个指定端口
5. 既不是非根桥的 RP, 也不是链路的 DP, 则这个端口为 block 端口

### 什么是最好的 BPDU?
按照如下规则选择
1. 最小的根桥RID
2. 最小的RPC
3. 最小的网桥 ID
4. 最小的PID


BPDU 转发
非根桥收到根桥发来的 BPDU 时, 会保留 RID, 换上自己的 BID, 换上自己的PID, 计算累计 PRC

在现网中, 可能会手动指定某一台设备的RID 为 0, 来保证后续进入网络的设备根桥不改变.
在稳定的生成树的网络中, 根端口负责解释根桥的 bpdu, 指定端口负责发送根桥的 BPDU
在连接主机的接口上, 因为从 PC 侧收到 BPDU, 所以自己的端口一定是指定端口. 在默认情况下会向 PC 发送 BPDU
